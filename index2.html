Git Befehlsübersicht

Git ist ein mächtiges Werkzeug zur Versionskontrolle. Hier sind einige der wichtigsten Befehle für den täglichen Gebrauch:

Setup & Initialisierung

git config --global user.name "Dein Name"

Setzt den Namen, der für alle deine Commits global verwendet wird.

git config --global user.email "deine.email@example.com"

Setzt die E-Mail-Adresse, die für alle deine Commits global verwendet wird.

git init

Initialisiert ein neues, leeres Git-Repository im aktuellen Verzeichnis.

git clone <repository_url>

Erstellt eine lokale Kopie eines entfernten Repositories (z.B. von GitHub).

Grundlegender Workflow

git status

Zeigt den Status der Dateien im Arbeitsverzeichnis und im Staging-Bereich an (geänderte, neue, gestagete Dateien).

git add <datei> oder git add .

Fügt Änderungen einer Datei (oder aller Dateien mit .) zum Staging-Bereich hinzu ("vormerken" für den nächsten Commit).

git commit -m "Deine Commit-Nachricht"

Speichert die Änderungen aus dem Staging-Bereich dauerhaft im Repository mit einer beschreibenden Nachricht.

git log

Zeigt die Commit-Historie des aktuellen Branches an.

Branching & Merging

git branch

Listet alle lokalen Branches auf. Der aktuell aktive Branch ist mit einem * markiert.

git branch <branchname>

Erstellt einen neuen Branch basierend auf dem aktuellen Commit.

git checkout <branchname> oder git switch <branchname>

Wechselt zum angegebenen Branch und aktualisiert das Arbeitsverzeichnis.

git checkout -b <branchname> oder git switch -c <branchname>

Erstellt einen neuen Branch und wechselt sofort zu ihm.

git merge <branchname>

Führt die Änderungen vom `` in den aktuellen Branch zusammen (Merge).

Arbeiten mit Remotes (z.B. GitHub)

git remote -v

Zeigt die konfigurierten entfernten Repositories (Remotes) mit ihren URLs an.

git fetch <remote_name>

Lädt alle Änderungen (neue Commits, Branches) vom Remote-Repository herunter, führt sie aber noch NICHT mit deiner lokalen Arbeit zusammen.

git pull <remote_name> <branchname>

Lädt Änderungen vom Remote herunter UND versucht, sie automatisch mit deinem aktuellen lokalen Branch zusammenzuführen (entspricht `git fetch` + `git merge`).

git push <remote_name> <branchname>

Lädt deine lokalen Commits und Branches zum Remote-Repository hoch.

Untersuchen & Vergleichen

git diff

Zeigt die Unterschiede zwischen deinem Arbeitsverzeichnis und dem Staging-Bereich an (Änderungen, die noch nicht mit `git add` hinzugefügt wurden).

git diff --staged oder git diff --cached

Zeigt die Unterschiede zwischen dem Staging-Bereich und dem letzten Commit an (Änderungen, die mit `git add` hinzugefügt, aber noch nicht committed wurden).

git show <commit_hash>

Zeigt detaillierte Informationen zu einem bestimmten Commit an (Autor, Datum, Nachricht, geänderte Dateien).

Viel Erfolg beim Arbeiten mit Git!
